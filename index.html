About Me

I love game design because it lets you be anything you want to be and live thousands of lives. Games are more than mechanics and levels, they’re worlds we step into, choices we make, and stories we share.

My passion is taking an idea, whether it’s a system, a space, or a narrative hook and turning it into something players can feel. I thrive on experimenting, prototyping, and iterating until the gameplay clicks and sparks that sense of discovery.

Whether I’m building procedural tools, crafting puzzle systems, or designing encounters, my goal is the same: to create immersive, meaningful, and memorable experiences that players carry with them long after the controller is down.

Project: Runtime Procedural World (BSc Dissertation)

PCG | C++ | UE5

Role: Solo project — designed, implemented, tested all systems.

Built a procedural world generation system in Unreal Engine 5 (C++/Blueprints) for my final-year dissertation, focused on runtime terrain creation.

Key Features:

  • Procedural terrain mesh using Perlin Noise for varied, natural landscapes
  • Tile-based Level of Detail (LOD) system for async streaming
  • Dynamic foliage placement driven by terrain rules for natural biomes
  • Spawn system ensuring players start in safe, navigable areas

Challenges and Solutions:

  • Overcame performance bottlenecks by implementing an efficient LOD system and asynchronous processing.
  • Ensured the terrain generation was visually compelling and topographically diverse through advanced procedural techniques.
  • Integrated foliage and environmental assets dynamically to maintain an immersive and realistic game world.
GitHub repo | Assignment Report

Project: Infinite Map Procedural Terrain Generator

Game Engine Development | C++ | Perlin Noise

Role: Solo project — responsible for design, code, and optimisation.

Developed a C++ engine subsystem to generate infinite procedural terrain using Perlin Noise.

Key Features:

  • Chunk-based world streaming for endless map generation.
  • Optimised rendering pipeline to lower memory usage and maintain FPS
  • Dynamic texturing by height (land/water differentiation) for visual clarity

Challenges and Solutions:

  • Memory Management: Addressed high memory usage by implementing an efficient Level of Detail (LOD) system and optimising the terrain rendering process.
  • Visual Quality: Improved the visual quality of the terrain by incorporating textures and optimising the Perlin Noise algorithm.
GitHub repo | Assignment Report

Project: Multiplayer Pong Game

SDL | C++ | Java-based Multiplayer Server

Reimagined classic Pong as a multiplayer SDL game with a Java TCP/IP server.

Key Features:

  • Three-player mode for added strategy and complexity
  • Power-ups altering ball physics to change gameplay dynamics
  • TCP/IP networking with client-side prediction for smooth synchronisation
  • Interactive menu and basic particle system for arcade polish

Challenges and Solutions:

  • Latency Management: Implemented client-side prediction and data normalisation to minimise latency and ensure responsive gameplay.
  • Error Handling: Developed robust error handling mechanisms to manage network-related issues and maintain game stability.
  • Scalability:Designed the architecture to handle multiple clients efficiently, with considerations for future expansions and additional features.
GitHub repo (Server) | GitHub repo (Client) | Assignment Report

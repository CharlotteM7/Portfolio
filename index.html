Project: Runtime Procedurally Generated World

PCG | C++ | UE5

For my final year dissertation, I developed a dynamic system for procedurally generating game worlds at runtime using Unreal Engine 5. This project focuses on creating diverse and engaging terrains using Perlin Noise, a method well-suited for generating smooth, natural-looking landscapes.


Key Features:

  • Terrain Generation: Implemented a procedural mesh system to generate terrains dynamically, ensuring unique landscapes for each playthrough.
  • Level of Detail (LOD): Developed a tile-based LOD system to optimise performance, rendering high-detail tiles close to the player and lower detail tiles further away.
  • Foliage and Environmental Assets: Integrated a system for dynamic foliage placement, ensuring that trees and plants are distributed naturally based on terrain features.
  • Performance Optimisation: Improved the gameâ€™s performance by managing terrain and asset generation asynchronously, maintaining smooth gameplay.
  • Player Interaction: Created a dynamic spawning system to ensure players always start in navigable areas, enhancing gameplay experience.

Challenges and Solutions:

  • Overcame performance bottlenecks by implementing an efficient LOD system and asynchronous processing.
  • Ensured the terrain generation was visually compelling and topographically diverse through advanced procedural techniques.
  • Integrated foliage and environmental assets dynamically to maintain an immersive and realistic game world.

Outcome:

This project successfully demonstrates the potential of procedural content generation to create rich, engaging, and performance-optimised game environments. It highlights the efficiency of using procedural techniques to manage large game worlds, ensuring unique and varied player experiences.

Project: Infinite Map Procedural Terrain Generator

Game Engine Development | C++ | Perlin Noise

For this project, I developed a game engine subsystem that generates infinite procedural terrain using Perlin Noise. This was implemented within a C++ codebase, focusing on creating an efficient and visually appealing terrain generation system.


Key Features:

  • Infinite Terrain Generation: Utilised a chunk-based system to generate an endless map, ensuring new terrain is procedurally created as the player moves.
  • Perlin Noise Algorithm: Implemented Perlin Noise for realistic terrain, balancing simplicity and performance.
  • Performance Optimisation: Enhanced rendering performance by optimising the terrain drawing process, reducing memory usage and ensuring smooth gameplay.
  • Dynamic Textures: Applied various textures based on terrain height to differentiate between land and water, improving visual appeal.

Challenges and Solutions:

  • Memory Management: Addressed high memory usage by implementing an efficient Level of Detail (LOD) system and optimising the terrain rendering process.
  • Visual Quality: Improved the visual quality of the terrain by incorporating textures and optimising the Perlin Noise algorithm.

Outcome:

This project showcases the ability to generate an infinite, procedurally created terrain in real-time. It highlights my skills in game engine development, procedural content generation, and performance optimisation.

Project: Multiplayer Pong Game

SDL | C++ | Java-based Multiplayer Server

For this project, I developed a multiplayer version of the classic Pong game, incorporating modern enhancements and additional features to create a more engaging experience. The development process involved both server-side and client-side programming, focusing on real-time gameplay and network communication.


Key Features:

  • Three-Player Mode: Expanded the classic Pong game to include a third player, adding a new dimension and strategic complexity to the gameplay.
  • Power-Up Mechanics: Introduced power-ups that alter the game's dynamics, such as slowing down the ball, which adds a strategic element to gameplay.
  • Particle System: Implemented a basic particle system to enhance visual effects, adding a retro arcade feel to the game.
  • Interactive Menu: Developed an interactive menu system for game settings and navigation, improving the user experience.
  • Networking and Synchronisation: Utilised TCP/IP for real-time data transmission, ensuring smooth and synchronised gameplay across multiple clients.

Challenges and Solutions:

  • Latency Management: Implemented client-side prediction and data normalisation to minimise latency and ensure responsive gameplay.
  • Error Handling: Developed robust error handling mechanisms to manage network-related issues and maintain game stability.
  • Scalability: Designed the architecture to handle multiple clients efficiently, with considerations for future expansions and additional features.

Outcome:

This project demonstrates my ability to develop and enhance classic games with modern features, focusing on real-time multiplayer capabilities. It showcases skills in game design, network programming, and performance optimisation.

  • Health and Healing System: Integrated a health system that allows the player to pick up health items to heal, enhancing gameplay dynamics.

  • Animation Synchronisation: Ensured smooth transitions between different animations by meticulously setting up blend spaces and state machines.

About Me

I love game design because it lets you be anything you want to be and live thousands of lives. Games are more than mechanics and levels, they’re worlds we step into, choices we make, and stories we share.

My passion is taking an idea, whether it’s a system, a space, or a narrative hook and turning it into something players can feel. I thrive on experimenting, prototyping, and iterating until the gameplay clicks and sparks that sense of discovery.

Whether I’m building procedural tools, crafting puzzle systems, or designing encounters, my goal is the same: to create immersive, meaningful, and memorable experiences that players carry with them long after the controller is down.

Project: Educational Cryptography Puzzle Game (MSc Dissertation)

UE5 | C++ | Blueprints | Systems & Puzzle Design

Role: Solo project — systems, puzzle design, and implementation.

A third-person educational puzzle game that teaches core cryptography concepts through clue-based progression and interactive puzzles.

Key Features:

  • Modular puzzle systems (substitution, pattern matching, decoding).
  • Quest and interaction systems built in C++ and Blueprints.
  • Data-driven puzzle logic so content can be tuned or extended without code changes.
  • UI feedback, hint systems, and basic accessibility adjustments.

Design Challenges & Solutions:

  • Balanced learning vs. frustration by iterating on puzzle difficulty curves.
  • Ensured puzzles teach real concepts (e.g. substitution ciphers) rather than feeling arbitrary.
  • Separated puzzle data from logic to make design iteration faster and safer.

Project: Runtime Procedural World (BSc Dissertation)

PCG | C++ | UE5

Role: Solo project — designed, implemented, tested all systems.

Procedural world generation system in Unreal Engine 5 (C++/Blueprints), focused on runtime terrain creation and streaming.

Key Features:

  • Procedural terrain meshes using Perlin Noise for varied, natural landscapes.
  • Tile-based Level of Detail (LOD) system for asynchronous world streaming.
  • Dynamic foliage placement driven by terrain rules for natural biomes.
  • Spawn system ensuring players start in safe, navigable areas.

Challenges and Solutions:

  • Reduced hitches by pushing work into async tasks and carefully managing LOD transitions.
  • Used biome rules to keep terrain visually interesting instead of random noise “blobs”.
  • Integrated foliage and environment assets dynamically to keep the world feeling cohesive.

Project: UE5 Puzzle-Solving Shooter Action Game

UE5 | Blueprints | Game Design | AI Behaviour Trees | Animation Blend Spaces

Role: Solo project — systems design, Blueprint scripting, and AI behaviours.

Unreal Engine 5 shooter prototype that combines environmental puzzles with combat, focusing on moment-to-moment feel and AI behaviour.

  • Shooting mechanics including ammo, reloading, and projectile behaviour.

Challenges and Solutions:

  • Improved AI reliability via UE perception tuning and behaviour tree debugging.
  • Smoothed animation transitions by carefully configuring blend spaces and state machine rules.
  • Made puzzles readable in combat spaces using lighting, affordances, and HUD signposting.

Project: Puzzle Solver App

React | Web Workers | three.js

Role: Group project — algorithms, performance, and a chunk of UI work.

A multi-solver interface capable of solving N-Queens, Polyspheres, and pyramid puzzles, with visualisation and performance-focused architecture.

Key Features:

  • Recursive solving algorithms with pruning and optimisation.
  • Heavy computation offloaded to multi-threaded Web Workers to keep the UI responsive.
  • 3D visualisation of puzzle states using three.js.
  • UI for switching solver types and configuring parameters.

Technical Focus:

  • Separated core solving logic from React components for testability and reuse.
  • Used worker messaging patterns to keep communication predictable and debuggable.
  • Handled long-running solves gracefully with loading states and progress feedback.
